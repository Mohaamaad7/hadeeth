# 🧠 تطوير محلل الأحاديث (HadithParser Refactoring)

## تاريخ التنفيذ
2026-03-01

## الملف
`app/Services/HadithParser.php`

---

## المشكلة
كان الـ `HadithParser` يعتمد على قاموس مصادر **ثابت (hardcoded)** في الكود:
```php
private array $sourceMap = [
    'خ' => 'صحيح البخاري',
    'م' => 'صحيح مسلم',
    // ...
];
```

عندما يُضاف مصدر جديد من لوحة التحكم (مثل "الطيالسي")، الـ Parser لا يتعرف عليه لأنه غير موجود في القاموس الثابت.

### المشكلة التقنية الأعمق
دالة `isExplanatoryParenthesis()` كانت ترفض أي كلمة طولها أكبر من 4 حروف ولم تكن موجودة في `$sourceMap`:
```php
// السطر 347 (قبل التعديل)
if (mb_strlen($part, 'UTF-8') > 4 && !isset($this->sourceMap[$part])) {
    return true; // ← يعتبرها نص تفسيري مش رمز مصدر!
}
```
يعني "الطيالسي" (7 حروف) كانت تُعامل كنص تفسيري وتُتجاهل.

## الحل

### التعديل: Constructor يحمّل المصادر من قاعدة البيانات

```php
class HadithParser
{
    private array $sourceMap;

    public function __construct()
    {
        // القاموس الثابت كـ fallback
        $fallback = [
            'خ' => 'صحيح البخاري',
            'م' => 'صحيح مسلم',
            'مالك' => 'موطأ الإمام مالك',
            'ط' => 'موطأ الإمام مالك',
            'حم' => 'مسند أحمد',
            // ... (كل المصادر الأصلية)
        ];

        // دمج مع المصادر من قاعدة البيانات
        try {
            $dbSources = \App\Models\Source::pluck('name', 'code')->toArray();
            $this->sourceMap = array_merge($fallback, $dbSources);
        } catch (\Throwable) {
            // إذا قاعدة البيانات غير متاحة (مثلاً أثناء migrations)
            $this->sourceMap = $fallback;
        }
    }
}
```

### لماذا `array_merge` وليس فقط DB؟
- **المصادر من DB تأخذ الأولوية** (لأنها ثانية في `array_merge`)
- **القاموس الثابت يعمل كـ fallback** إذا قاعدة البيانات فارغة أو غير متاحة
- **مصادر إضافية في القاموس** مثل `'ط' => 'موطأ الإمام مالك'` (رموز بديلة) لا يلزم إضافتها في DB

### لماذا `try/catch`؟
أثناء تشغيل `php artisan migrate` لأول مرة، جدول `sources` قد لا يكون موجوداً بعد، فالـ `catch` يمنع الكراش.

## بنية الـ Parser الكاملة

### الدوال الرئيسية:

| الدالة | الوظيفة | المدخل | المخرج |
|---|---|---|---|
| `parse(string $text)` | تحليل حديث واحد | النص الأصلي الكامل | مصفوفة بالبيانات المستخرجة |
| `parseMultiple(string $bulkText)` | تحليل نصوص متعددة | نص يحتوي أحاديث | مصفوفة من الأحاديث المحللة |

### خطوات `parse()`:
```
النص الأصلي
    ↓
1. extractNumber()      → [4193]
2. extractGrade()       → (صحيح)
3. extractAdditions()   → زاد (طب) في آخره: ...
4. extractNarrator()    → عن جابر
5. extractSourceCodes() → (الطيالسي حم م)
6. decodeSources()      → أسماء المصادر الكاملة
7. cleanText()          → نص الحديث النظيف
    ↓
{ number, grade, narrator, source_codes, sources, additions, clean_text }
```

### دالة `extractSourceCodes()` — التفصيل:

```
النص: (الطيالسي حم م)
    ↓
1. استخراج محتوى كل قوسين ()
2. تخطي الدرجات: (صحيح), (حسن)
3. تخطي النصوص التفسيرية: (يعني الوحي)
4. فحص مجموعات التوسع: ق → [خ, م]
5. تقسيم بالمسافات: ["الطيالسي", "حم", "م"]
6. لكل جزء:
   - فحص مجموعات التوسع
   - فحص sourceMap (القاموس المدمج)
   - إذا ≤ 3 حروف: فصل الحروف واحداً واحداً
    ↓
["الطيالسي", "حم", "م"]
```

### دالة `isExplanatoryParenthesis()` — التفصيل:

```
المدخل: محتوى الأقواس
    ↓
1. فحص كلمات تفسيرية: يعني، أي، يريد، قال، قوله
   → إذا وُجدت ← true (تفسيرية)
2. فحص طول الكلمات:
   → إذا كلمة > 4 حروف AND ليست في sourceMap ← true
   (الآن sourceMap يشمل DB فـ "الطيالسي" تمر ✅)
    ↓
false = رموز مصادر
true  = نص تفسيري (تجاهل)
```

### دالة `extractNarrator()`:
```
النمط: عن NARRATOR_NAME
التوقف عند: [ أو ( أو كلمات الزيادة أو نهاية النص أو نقطة

تطبيع الاسم (normalizeNarrator):
- أبي → أبو (إلا "أبي بن كعب")
- أبا → أبو
- إزالة النقطة الأخيرة
```

### دالة `extractAdditions()`:
```
الكلمات المفتاحية: زاد, وزاد, ولفظ, ورواه, وعند, وفي رواية
النمط: KEYWORD (SOURCE_CODE) CONTEXT: ADDITION_TEXT

مثال:
  زاد (طب) في آخره: وهو أهونه علي.
  → { source_code: "طب", source_name: "المعجم الكبير للطبراني", text: "وهو أهونه علي" }
```

### مجموعات التوسع (`groupExpansion`):
| الرمز | التوسع | المعنى |
|---|---|---|
| `ق` | `['خ', 'م']` | متفق عليه (البخاري ومسلم) |
| `4` | `['د', 'ت', 'ن', 'هـ']` | السنن الأربعة |
| `3` | `['د', 'ت', 'ن']` | السنن الثلاثة (بدون ابن ماجه) |

## أثر التغيير
- ✅ أي مصدر يُضاف من لوحة التحكم يُتعرف عليه فوراً
- ✅ لا حاجة لتعديل الكود عند إضافة مصدر جديد
- ✅ القاموس الثابت يبقى كـ fallback + يحتوي رموز بديلة (مثل `ط` للموطأ)
- ✅ `isExplanatoryParenthesis` تتعرف على المصادر ذات الأسماء الطويلة
