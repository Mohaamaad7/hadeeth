✅ تم الإصلاح SQL Injection — حقن SQL في البحث عن المصادر
HadithController.php (Dashboard)

تم حل المشكلة بتاريخ 4 مارس 2026. تم استبدال استعلامات LIKE الخام باستراتيجية مطابقة آمنة على مرحلتين:

  • تحميل جميع المصادر من قاعدة البيانات مرة واحدة (Source::all())
  • مطابقة تامة أولاً (firstWhere('name', $sourceName))
  • Fallback آمن في الذاكرة (str_contains) بدون أي SQL
  • حذف تكرار الـ IDs (in_array check)

قبل الإصلاح:

$source = Source::where('name', 'LIKE', "%{$sourceName}%")->first();
// ← SQL injection + N+1 query في كل تكرار

بعد الإصلاح:

$allSources = Source::all(); // تحميل مسبق مرة واحدة

// 1️⃣ مطابقة تامة (الحالة الطبيعية)
$source = $allSources->firstWhere('name', $sourceName);

// 2️⃣ Fallback آمن في الذاكرة
if (!$source) {
    $source = $allSources->first(fn(Source $s) => str_contains($s->name, $sourceName));
}

✅ تم تطبيق الإصلاح في store() (سطر 203) و bulkStore() (سطر 657)

✅ تم الإصلاح عمليات Cleanup بدون DB Transaction — خطر فقدان بيانات

تم حل المشكلة بتاريخ 4 مارس 2026. تم لف جميع عمليات الحذف المتعددة الجداول في DB::transaction() لضمان الذرية (atomicity).

  • deleteHadiths() — 5 جداول داخل transaction + إضافة hadith_narrator المفقود
  • deleteAllNarrators() — جدولين داخل transaction
  • deleteAllBooks() — جدولين داخل transaction
  • deleteChains() — جدولين داخل transaction
  • nukeAll() — 8 جداول داخل transaction + إضافة hadith_narrator المفقود

قبل الإصلاح:

DB::table('hadith_source')->delete();
DB::table('chain_narrators')->delete();
HadithChain::query()->delete();  // ❌ لو فشل هنا؟
Hadith::query()->delete();       // ← لن يتم تنفيذه!

بعد الإصلاح:

DB::transaction(function () {
    DB::table('hadith_source')->delete();
    DB::table('hadith_narrator')->delete(); // ← كان مفقوداً!
    DB::table('chain_narrators')->delete();
    HadithChain::query()->delete();
    Hadith::query()->delete();
}); // ← لو فشل أي شيء → rollback تلقائي

15 Narrator Model — علاقة hadiths() ناقصة العكس
Narrator.php

Narrator يعرّف hadiths() كـ HasMany (من narrator_id) لكن لا يوجد علاقة hadithsMany() كـ BelongsToMany عبر hadith_narrator. 

// Narrator.php الحالي:
public function hadiths(): HasMany  // ← من narrator_id فقط

// المفقود:
public function hadithsM2M(): BelongsToMany
{
    return $this->belongsToMany(Hadith::class, 'hadith_narrator');
}

هذا يتسبب في أن CleanupController::deleteOrphanNarrators() يفحص hadiths() (HasMany) ويعتبر الراوي يتيمًا حتى لو هو مرتبط عبر الجدول الوسيط M2M!
